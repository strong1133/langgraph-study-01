## 2. 채택 접근법 & 근거

기존 Git 이력과 LangChain 노트북을 복제하지 않고 동일한 개발 구조를 새 저장소에 재구성한다. 학습 주제별 이력을 분리하면서 개발 경험은 동일하게 유지하기 위해서다.

## 3. 변경 범위 & 파일

새 프로젝트 내부의 Python 프로젝트 파일, Cursor 설정, 빈 LangGraph 노트북, 개발 도구 설정만 만든다. 원본 프로젝트는 변경하지 않으며 `.git`, `.venv`, 캐시, `.env`는 복제하지 않는다.

## 4. 프로젝트 간 계약 영향

없음. 두 저장소는 독립적으로 동작한다.

## 5. 리스크 & 미해결 가정

`dev로 셋팅`은 개발환경 구성으로 해석하고 원본과 동일한 `main` 브랜치를 사용한다. GitHub 인증은 기존 로컬 자격증명을 안전하게 재사용한다.

`uv lock --check`, Ruff, Pyright, Pytest, 메인 프로그램 실행, Git 상태 및 GitHub API 조회를 수행한다.
